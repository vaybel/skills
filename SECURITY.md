# Security

## Reporting

Report security issues privately to `support@vaybel.com`. Do not open a public
GitHub issue for vulnerabilities, leaked tokens, or customer data exposure.

## Secrets

This repository must not contain live Vaybel PATs, customer artifacts, generated
mockups, or provider credentials. Use placeholders such as `<your-vaybel-pat>`
in documentation.

## Supported Version

Only the latest released version is supported.
